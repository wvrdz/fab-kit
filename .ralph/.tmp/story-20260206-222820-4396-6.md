### US-006: Create fab-new.md skill
Status: in_progress
Depends on: US-004

Description:
As a developer, I want the /fab:new skill so I can start a new change from a description.

Acceptance Criteria:
- [ ] Create fab/.kit/skills/fab-new.md as comprehensive agent-agnostic markdown
- [ ] Must reference _context.md preamble at the top
- [ ] Accepts <description> argument and optional --branch <name>
- [ ] Generates folder name: {YYMMDD}-{XXXX}-{slug} where YYMMDD is today, XXXX is 4 random lowercase alphanumeric, slug is 2-4 words from description (all lowercase)
- [ ] Creates fab/changes/{name}/ directory
- [ ] Writes change name to fab/current (sets active change)
- [ ] Git integration: if git.enabled in config and inside git repo — offer create branch, adopt current branch, or skip. If --branch provided, use that name directly. Record branch in .status.yaml
- [ ] Initializes .status.yaml from template with stage: proposal, all progress pending except proposal: active
- [ ] Generates proposal.md using fab/.kit/templates/proposal.md, loading constitution and config as context
- [ ] Asks clarifying questions if intent is ambiguous (max 3 [BLOCKING])
- [ ] Marks proposal complete once user is satisfied
- [ ] Output ends with: Next: /fab:continue or /fab:ff (fast-forward all planning)
- [ ] Example: /fab:new Add OAuth2 support → creates 260206-x7k2-add-oauth/
- [ ] Negative case: if fab/config.yaml doesn't exist, prompt user to run /fab:init first
