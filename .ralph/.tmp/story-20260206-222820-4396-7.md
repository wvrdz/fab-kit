### US-007: Create fab-continue.md skill
Status: in_progress
Depends on: US-004

Description:
As a developer, I want the /fab:continue skill so I can advance to the next stage or reset to a previous stage.

Acceptance Criteria:
- [ ] Create fab/.kit/skills/fab-continue.md as comprehensive agent-agnostic markdown
- [ ] Must reference _context.md preamble at the top
- [ ] Normal flow (no argument): reads .status.yaml, identifies next artifact, loads template + context (including constitution), generates artifact, updates .status.yaml
- [ ] Plan decision: when transitioning from specs to plan, evaluate if plan is warranted. If small/obvious, propose skipping to user. If agreed, record plan: skipped in .status.yaml
- [ ] Auto-generates checklist (checklists/quality.md) when creating tasks stage
- [ ] Reset flow (with stage argument): accepts specs, plan, or tasks. Cannot reset to proposal/apply/review/archive. Resets .status.yaml, regenerates target artifact in place, invalidates downstream (tasks reset to unchecked, checklist regenerated)
- [ ] Context loading varies by stage as specified in SKILLS.md
- [ ] Next steps vary by stage reached: specs done → 'Next: /fab:continue (plan) or /fab:ff or /fab:clarify', plan done → 'Next: /fab:continue (tasks) or /fab:clarify', tasks done → 'Next: /fab:apply'
- [ ] Example: after proposal done, /fab:continue creates spec.md
- [ ] Negative case: if no active change (fab/current missing), tell user to run /fab:new
