### US-013: Create fab-switch.md and fab-status.md skills
Status: in_progress
Depends on: US-004

Description:
As a developer, I want the switch and status skills so I can manage multiple changes and see progress.

Acceptance Criteria:
- [ ] Create fab/.kit/skills/fab-switch.md: accepts <change-name> argument, matches against fab/changes/ (supports partial/slug match), handles ambiguous match (list and ask), handles no match (list available), writes full name to fab/current, displays switched change status
- [ ] Create fab/.kit/skills/fab-status.md: shows current change state — change name, branch (if set), stage (N/7), progress table with symbols (checkmark=done, circle=active, empty=pending), checklist status, suggested next command
- [ ] Both must reference _context.md preamble at the top
- [ ] fab-status.md shows kit version from fab/.kit/VERSION
- [ ] fab-switch.md: when no argument provided, lists all active changes and asks user to pick
- [ ] Example: /fab:switch fix-checkout → 'fab/current now points to 260202-m3x1-fix-checkout-bug'
- [ ] Example: /fab:status → displays formatted progress table with stage 3/7
- [ ] Negative case for switch: partial match 'add' matches multiple changes → list all matches, ask user to pick
- [ ] Negative case for status: no active change → 'No active change. Run /fab:new to start one.'
