# Migration: 1.1.0 to 1.2.0

## Summary

Add `review_tools` configuration block to `fab/project/config.yaml`. This enables per-tool enable/disable for the cascading PR review tools feature in `/git-pr-review`.

## Pre-check

1. Confirm the project has a `fab/project/config.yaml` file — if not, skip this migration entirely

## Changes

### 1. Add `review_tools` block to config.yaml

Read `fab/project/config.yaml` and check if a `review_tools` key already exists.

If `review_tools` key **does not exist**:

1. Append the following block to `fab/project/config.yaml`:
   ```yaml
   review_tools:
       copilot: true
       codex: true
       claude: true
   ```
2. Print: `Added: review_tools block to fab/project/config.yaml (all tools enabled by default)`

If `review_tools` key **already exists**, print: `review_tools already present in fab/project/config.yaml, skipping.`

## Verification

1. `fab/project/config.yaml` contains a `review_tools` key with `copilot`, `codex`, and `claude` sub-keys
