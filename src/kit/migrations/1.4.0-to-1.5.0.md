# Migration: 1.4.0 to 1.5.0

## Summary

Restructure `.fab-runtime.yaml` so agents become first-class entries keyed by
Claude Code's `session_id` (a UUID from the hook stdin payload) instead of
per-change-folder `agent:` blocks. This unlocks `fab pane map` visibility for
agents running in discussion mode (before `/fab-new`) and cleanly separates
the three orthogonal axes the pane map tracks: **Change** (from
`.fab-status.yaml`), **Agent** (from `_agents`), and **Process** (opt-in via
`fab pane process`).

The old schema cannot be faithfully migrated — previous hooks never captured
`session_id`, so there is no stable identity to key the new entries on.
Strategy is **clean-slate**: delete any existing `.fab-runtime.yaml` at each
worktree's repo root. Runtime state is ephemeral and re-populates on the
next hook event (Stop / SessionStart / UserPromptSubmit).

**Transient effect**: immediately after running this migration, `fab pane
map` may show `—` in the Agent column for up to one hook cycle until the
next Claude event fires and repopulates the file.

## Pre-check

1. Confirm the current directory is a git repository — if not, skip this
   migration entirely.

## Changes

### 1. Delete `.fab-runtime.yaml` in the current worktree

If `.fab-runtime.yaml` exists at the repo root of the current worktree:

1. Delete the file.
2. Print: `Removed: .fab-runtime.yaml at <repo-root> (schema restructured to session-id keying; state will re-populate on next hook event).`

If `.fab-runtime.yaml` does not exist at the repo root, print:
`.fab-runtime.yaml already absent at <repo-root>, skipping.`

### 2. Delete `.fab-runtime.yaml` in all sibling worktrees

Run `git worktree list --porcelain` to enumerate all worktrees tracked by the
current repository. For each worktree path reported (other than the current
one, which was handled above):

1. If `<worktree-path>/.fab-runtime.yaml` exists, delete it and print:
   `Removed: .fab-runtime.yaml at <worktree-path>.`
2. Otherwise, skip silently.

## Verification

1. `.fab-runtime.yaml` does NOT exist at the current worktree's repo root.
2. For each worktree in `git worktree list --porcelain`, its
   `.fab-runtime.yaml` (if any existed before this migration) is gone.
3. On the next Stop / SessionStart / UserPromptSubmit hook event, a new
   `.fab-runtime.yaml` is written with the `_agents[<session_id>]` schema.
