# Project Context

Tech stack: Markdown specs, shell scripts (Bash), Claude Code skills
Content: Research analyses of SpecKit and OpenSpec, cross-cutting findings
Workflow: Fab kit for specification-driven change management
Structure: Monorepo with doc/, fab/ directories
Local backlog: fab/backlog.md
Distribution: Only fab/.kit/ folder is distributed to the user. fab/project is created for the user, and can refer to fab/.kit, but not anything else.
Skills: Canonical skill sources live at `fab/.kit/skills/*.md`. The files in `.claude/skills/` are deployed copies generated by `fab-sync.sh`. When modifying a skill, always update the source at `fab/.kit/skills/` — never edit only the deployed copy, or the change will be overwritten on next sync.
Migrations: When a change requires restructuring existing user data (config files, `.status.yaml`, archive layout, etc.), it MUST ship as a migration file in `fab/.kit/migrations/` — not as a new subcommand or ad-hoc script. Migrations are markdown instruction files applied by `/fab-setup migrations`. See `docs/memory/fab-workflow/migrations.md` for the format and versioning model.
