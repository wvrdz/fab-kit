# Migration: 0.9.0 to 0.10.0

## Summary

Move the 5 Cs (constitution.md, context.md, code-quality.md, code-review.md, config.yaml) and VERSION file from `fab/` into a new `fab/project/` subfolder. This creates a clean top-level hierarchy separating project identity files from workflow machinery.

## Pre-check

- At least one of the 6 files exists at the old `fab/` location: `config.yaml`, `constitution.md`, `context.md`, `code-quality.md`, `code-review.md`, `VERSION`
- If all 6 are already at `fab/project/` (or absent from both locations), skip — already migrated

## Changes

### 1. Create `fab/project/` directory

If `fab/project/` does not exist, create it:

```bash
mkdir -p fab/project
```

### 2. Move files from `fab/` to `fab/project/`

For each of the following files, move from `fab/{file}` to `fab/project/{file}`:

1. `config.yaml`
2. `constitution.md`
3. `context.md`
4. `code-quality.md`
5. `code-review.md`
6. `VERSION`

**Rules for each file:**
- If the file exists at `fab/{file}` and does NOT exist at `fab/project/{file}`: move it (`mv fab/{file} fab/project/{file}`)
- If the file exists at both locations: skip (preserve the `fab/project/` copy, warn about the duplicate at `fab/`)
- If the file does not exist at `fab/{file}`: skip (optional files like `context.md`, `code-quality.md`, `code-review.md` may not exist)

### 3. Update config.yaml header comment

If `fab/project/config.yaml` contains `# fab/config.yaml` as the first line, replace it with `# fab/project/config.yaml`.

## Verification

- `fab/project/config.yaml` exists
- `fab/project/constitution.md` exists
- None of the 6 files exist at the old `fab/` location (unless they were absent to begin with)
- `fab/project/VERSION` exists (if it existed before migration)
