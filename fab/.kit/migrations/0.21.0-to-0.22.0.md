# Migration: 0.21.0 to 0.22.0

## Summary

Remove the `model_tiers:` section from `fab/project/config.yaml`. The fast model tier has been eliminated — all skills now run on the session's default model. The `model_tiers:` configuration is no longer read by any code.

## Pre-check

- `fab/project/config.yaml` exists
- `fab/project/config.yaml` contains a `model_tiers:` key
- If no `model_tiers:` key exists: skip — already migrated or never had one

## Changes

### 1. Remove `model_tiers:` section

Remove the entire `model_tiers:` block from `fab/project/config.yaml`, including any preceding comment line:

```bash
yq -i 'del(.model_tiers)' fab/project/config.yaml
```

If there is a comment line immediately before `model_tiers:` (e.g., `# Maps tier names to...`), remove that comment line as well.

### 2. Bump version

```bash
echo "0.22.0" > fab/.kit-migration-version
```

## Verification

- `fab/project/config.yaml` does NOT contain a `model_tiers:` key: `yq '.model_tiers' fab/project/config.yaml` returns `null`
- All other config sections are preserved: `project`, `source_paths`, `checklist`, `stage_directives`
- `fab/.kit-migration-version` contains `0.22.0`
