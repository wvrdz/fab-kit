# Migration: 0.22.0 to 0.24.0

## Summary

Consolidate `.status.yaml` field naming: `issue_id` (scalar) → `issues` (array), `shipped` (array) → `prs` (array). Rename `.shipped` sentinel file to `.pr-done`. Applies to active (non-archived) changes only.

## Pre-check

- `fab/changes/` directory exists
- At least one non-archived change exists with `.status.yaml`, OR no active changes (migration is a no-op for status files but still bumps version)

## Changes

### 1. Migrate active `.status.yaml` files

For each `fab/changes/*/.status.yaml` (excluding `fab/changes/archive/`):

```bash
for status_file in fab/changes/*/.status.yaml; do
  [ -f "$status_file" ] || continue
  # Skip archive
  case "$status_file" in fab/changes/archive/*) continue ;; esac

  # Convert issue_id (scalar or null) to issues (array)
  issue_val=$(yq '.issue_id // ""' "$status_file")
  if [ -n "$issue_val" ] && [ "$issue_val" != "null" ]; then
    yq -i ".issues = [\"${issue_val}\"] | del(.issue_id)" "$status_file"
  else
    yq -i '.issues = [] | del(.issue_id)' "$status_file"
  fi

  # Rename shipped to prs (preserve contents)
  yq -i '.prs = (.shipped // []) | del(.shipped)' "$status_file"
done
```

### 2. Rename sentinel files

```bash
for sentinel in fab/changes/*/.shipped; do
  [ -f "$sentinel" ] || continue
  case "$sentinel" in fab/changes/archive/*) continue ;; esac
  mv "$sentinel" "$(dirname "$sentinel")/.pr-done"
done
```

### 3. Bump version

```bash
echo "0.24.0" > fab/.kit-migration-version
```

## Verification

- No active `.status.yaml` contains `issue_id` or `shipped` keys: `yq '.issue_id // .shipped' fab/changes/*/.status.yaml` returns null for all
- Active `.status.yaml` files contain `issues` and `prs` keys (both arrays)
- No `.shipped` sentinel files exist under active changes
- `fab/.kit-migration-version` contains `0.24.0`
