# Migration: 0.38.0 to 0.39.0

## Summary

Add the `agent` section with `spawn_command` to `fab/project/config.yaml`. This key controls which command scripts use to spawn agent sessions (operators, batch scripts).

## Pre-check

1. Confirm `fab/.kit/VERSION` reads `0.39.0` or higher (kit has been upgraded)
2. Check if `fab/project/config.yaml` already contains an `agent:` section — if so, skip with a note that it already exists

## Changes

### 1. Add `agent` section to `config.yaml`

If `fab/project/config.yaml` does not contain an `agent:` line:

1. Append the following block to the end of `fab/project/config.yaml`:
   ```yaml

   agent:
     # Base command for spawning agent sessions in scripts and operators.
     # Shell expansions (e.g., $(basename "$(pwd)")) expand at invocation time.
     spawn_command: 'claude --dangerously-skip-permissions --effort max -n "$(basename "$(pwd)")"'
   ```
2. Print a note:
   ```
   Added agent.spawn_command to fab/project/config.yaml.
   ```

If `fab/project/config.yaml` already contains `agent:`:

1. Print a note:
   ```
   agent section already exists in config.yaml, skipping.
   ```

## Verification

1. `fab/project/config.yaml` should contain an `agent:` section with a `spawn_command` key
2. Running `yq -r '.agent.spawn_command' fab/project/config.yaml` should return a non-empty string
