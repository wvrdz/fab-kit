# Migration: 0.34.0 to 0.37.0

## Summary

Surface the new optional `linear_workspace` config field to existing users. When configured, `/git-pr` uses this field to render Linear issue IDs as clickable links in PR descriptions.

## Pre-check

1. Confirm `fab/.kit/VERSION` reads `0.37.0` or higher (kit has been upgraded)
2. Check if `fab/project/config.yaml` exists — if not, skip this migration entirely (config absence is a pre-existing issue)
3. Check if `fab/project/config.yaml` already contains a `linear_workspace` field — if so, skip with a note that the field already exists

## Changes

### 1. Add commented-out `linear_workspace` field

If `fab/project/config.yaml` exists and does not contain `linear_workspace`:

1. Locate the `project:` block in `fab/project/config.yaml`
2. Add the following commented-out line after the last existing field under `project:` (typically after `description:`):
   ```yaml
   # linear_workspace: "your-workspace"  # optional — enables Linear issue links in PRs
   ```
3. Print a note:
   ```
   Added commented-out linear_workspace field to fab/project/config.yaml.
   To enable Linear issue links in PRs, uncomment and set your workspace slug:
     linear_workspace: "your-workspace"
   The workspace slug is the subdomain in your Linear URL: https://linear.app/{workspace}/
   ```

## Verification

1. `fab/project/config.yaml` should contain the commented-out `linear_workspace` line under the `project:` block
2. No behavioral changes — the field is commented out and `/git-pr` treats its absence as a no-op
