# Migration: 0.24.0 to 0.32.0

## Summary

Archive layout changed from flat `archive/{name}/` to date-bucketed `archive/yyyy/mm/{name}/`. Existing flat archive entries need to be moved into the new structure. The `yyyy` and `mm` are derived from each folder's `YYMMDD` prefix (first 6 characters).

## Pre-check

- `fab/changes/archive/` directory exists (if not, skip — no archive to migrate)
- At least one flat entry exists directly under `archive/` (directories whose names don't match `^[0-9]{4}$`)

## Changes

### 1. Move flat archive entries into date buckets

For each directory directly under `fab/changes/archive/` that is not a 4-digit year directory:

```bash
archive_dir="fab/changes/archive"
for d in "$archive_dir"/*/; do
  [ -d "$d" ] || continue
  base="$(basename "$d")"
  # Skip year directories (already bucketed structure)
  [[ "$base" =~ ^[0-9]{4}$ ]] && continue

  # Parse YYMMDD prefix
  yy="${base:0:2}"
  mm="${base:2:2}"
  dest_dir="$archive_dir/20${yy}/${mm}"

  # Skip if already at destination (idempotent)
  [ -e "$dest_dir/$base" ] && continue

  mkdir -p "$dest_dir"
  mv "$d" "$dest_dir/$base"
done
```

### 2. Bump version

```bash
echo "0.32.0" > fab/.kit-migration-version
```

- Archived changes are at `fab/changes/archive/yyyy/mm/{name}/`

- `fab/.kit-migration-version` contains `0.32.0`
