# Migration: 0.20.0 to 0.21.0

## Summary

Rename the migration tracking file from `fab/project/VERSION` to `fab/.kit-migration-version`. The new name is clearer about its purpose and sits alongside the new `fab/.kit-sync-version` file with a consistent naming pattern.

## Pre-check

- At least one of these exists:
  - `fab/project/VERSION` (old location)
  - `fab/.kit-migration-version` (new location, if `fab-sync.sh` already migrated it)
- If neither exists: skip — nothing to migrate

## Changes

### 1. Move `fab/project/VERSION` to `fab/.kit-migration-version`

If `fab/project/VERSION` exists and `fab/.kit-migration-version` does not:

```bash
mv fab/project/VERSION fab/.kit-migration-version
```

If `fab/project/VERSION` exists and `fab/.kit-migration-version` also exists (edge case — sync already created it):

```bash
rm fab/project/VERSION
```

If only `fab/.kit-migration-version` exists (sync already migrated): skip the move.

### 2. Bump version

```bash
echo "0.21.0" > fab/.kit-migration-version
```

## Verification

- `fab/.kit-migration-version` contains `0.21.0`
- `fab/project/VERSION` does NOT exist
