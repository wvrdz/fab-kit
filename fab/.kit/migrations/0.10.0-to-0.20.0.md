# Migration: 0.10.0 to 0.20.0

## Summary

Simplify `config.yaml`: remove `git` and `naming` sections, rename `rules` → `stage_directives` with all-stage placeholders, strip verbose comments. Add `issue_id` field to status.yaml template.

## Pre-check

- `fab/project/config.yaml` exists
- At least one of these conditions is true:
  - `config.yaml` contains a `git:` key
  - `config.yaml` contains a `naming:` key
  - `config.yaml` contains a `rules:` key (not yet renamed to `stage_directives`)
- If none of the above, skip — already migrated

## Changes

### 1. Remove `git:` block

If `config.yaml` contains a `git:` section, remove it entirely (the key and all nested content up to the next top-level key).

```bash
yq -i 'del(.git)' fab/project/config.yaml
```

### 2. Remove `naming:` block

If `config.yaml` contains a `naming:` section, remove it entirely.

```bash
yq -i 'del(.naming)' fab/project/config.yaml
```

### 3. Rename `rules:` → `stage_directives:`

If `config.yaml` contains a `rules:` key, rename it to `stage_directives:` and ensure all six stage placeholders exist:

```bash
yq -i '.stage_directives = .rules | del(.rules)' fab/project/config.yaml
```

Then add empty arrays for any missing stages:

```bash
for stage in intake spec tasks apply review hydrate; do
  yq -i ".stage_directives.$stage //= []" fab/project/config.yaml
done
```

### 4. Strip verbose comments and align with scaffold

Replace the file's comments to match the reduced-verbosity style of the updated scaffold at `fab/.kit/scaffold/fab/project/config.yaml`. Preserve all project-specific values (project name, description, source_paths, extra_categories, stage directives, model_tiers).

The resulting file should have:
- No file header comment block
- No multi-line `# Consumed by:` blocks
- Single-line comments per field
- Top-level keys only: `project`, `source_paths`, `checklist`, `stage_directives`, `model_tiers`

### 5. Bump VERSION

```bash
echo "0.20.0" > fab/.kit-migration-version
```

## Verification

- `fab/project/config.yaml` does NOT contain `git:` key: `yq '.git' fab/project/config.yaml` returns `null`
- `fab/project/config.yaml` does NOT contain `naming:` key: `yq '.naming' fab/project/config.yaml` returns `null`
- `fab/project/config.yaml` does NOT contain `rules:` key: `yq '.rules' fab/project/config.yaml` returns `null`
- `fab/project/config.yaml` contains `stage_directives:` with all 6 stages: `yq '.stage_directives | keys' fab/project/config.yaml` returns all 6
- `fab/.kit-migration-version` contains `0.20.0`
- Project-specific values are preserved (project.name, source_paths, extra_categories, model_tiers)
