# Intake: Operator7 Direct fab-new Spawn

**Change**: 260326-13ro-operator7-direct-fab-new-spawn
**Created**: 2026-03-26
**Status**: Draft

## Origin

> Update fab-operator7.md skill: operator should pass the task description directly to fab-new when spawning agents, not create a backlog entry first with idea add. Remove the idea add step from the spawn sequence.

One-shot request. The user wants to simplify the operator's "from raw text" spawn path by removing the intermediate `idea add` step.

## Why

The current operator7 "from raw text" flow creates an unnecessary indirection: raw text → `idea add` → backlog entry → `/fab-new <id>`. Since `/fab-new` already accepts natural language descriptions directly, the backlog entry is redundant overhead. It creates a backlog artifact that serves no further purpose — the change folder and intake are the real artifacts of record.

If unchanged, every ad-hoc text request routed through the operator creates an orphaned backlog entry that adds noise to `fab/backlog.md` without providing traceability value (the intake's Origin section already captures the raw input).

## What Changes

### Remove `idea add` from "From raw text" spawn path

In `fab/.kit/skills/fab-operator7.md`, section §6 "Working a Change", the **"From raw text"** subsection currently reads:

```markdown
**From raw text** (e.g., "fix login after password reset"):
1. Create backlog entry: `idea add "<description>"` — captures the ID
2. Proceed with the structured flow above using the new ID
```

Replace with a direct `/fab-new` invocation that passes the description as-is:

```markdown
**From raw text** (e.g., "fix login after password reset"):
1. Create worktree (`wt create --non-interactive`)
2. Spawn agent: `tmux new-window -n "fab-<wt>" -c <worktree-path> "<spawn_cmd> '/fab-new <description>'"`
3. Enroll in monitored set
4. On completion: merge PR, optionally archive
```

This aligns with the "From backlog ID or Linear issue" flow but skips the backlog lookup since there's no ID to resolve. The `/fab-new` skill handles slug generation, folder creation, and intake generation from the raw description.

### Remove explanatory paragraph about `idea add`

The paragraph after "From raw text" that reads:

> This ensures every change gets a proper intake artifact with traceability, even for ad-hoc requests. The operator handles `idea add` internally — the user just says "fix [description]" and the operator does the rest.

Should be updated to remove the `idea add` reference since the traceability argument still holds — `/fab-new` creates the intake with the Origin section capturing the raw input. The updated paragraph should explain that `/fab-new` handles traceability directly.
