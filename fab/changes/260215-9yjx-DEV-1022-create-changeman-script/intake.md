# Intake: Create changeman.sh and Refactor fab-new Skill

**Change**: 260215-9yjx-DEV-1022-create-changeman-script
**Created**: 2026-02-15
**Status**: Draft

## Origin

> Create changeman.sh to consolidate fab-new's filesystem boilerplate (random ID generation, mkdir, .status.yaml template filling, created_by detection, stageman calls) into a single shell script. Refactor fab-new skill to delegate the mechanical work and focus on AI-driven tasks (slug generation, SRAD, intake writing). Designed CLI-first for eventual Rust rewrite parity with stageman.

## Why

The fab-new skill currently performs 5+ separate bash operations inline — `/dev/urandom` for ID generation, `mkdir` for folder creation, `gh api user`/`git config` for created_by, template substitution via `sed`, and two `stageman.sh` CLI calls. Each triggers a separate Claude Code permission prompt, creating unnecessary friction for a deterministic, mechanical operation. Consolidating into a single `changeman.sh new` call reduces this to one prompt and establishes a clean boundary: scripts handle plumbing, skills handle judgment.

## What Changes

- **New `fab/.kit/scripts/lib/changeman.sh`** with `new` subcommand:
  - Interface: `changeman.sh new --slug <slug> [--change-id <4char>] [--log-args <description>]`
  - Generates date (YYMMDD), random or provided 4-char ID, constructs folder name
  - Collision detection (fatal for provided IDs, retry for random)
  - Creates directory, fills `.status.yaml` from template via `sed`, detects `created_by`
  - Calls `stageman.sh set-state` and `stageman.sh log-command` via CLI (not sourced)
  - Prints folder name to stdout; errors to stderr
- **Refactored `fab/.kit/skills/fab-new.md`**:
  - Steps 1, 3, 4 collapse — Step 1 narrows to "Generate Slug" (AI task), Steps 3-4 become a single `changeman.sh new` call
  - Error handling table simplified — collision/creation entries become one "changeman.sh failure" entry
  - Renumbered steps (7 → 6)

## Affected Memory

- `fab-workflow/planning-skills`: (modify) Reflect changeman.sh as the initialization mechanism for fab-new

- **No existing functionality removed** — the same operations happen, just consolidated into one script
